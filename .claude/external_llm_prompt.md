# Swaya.me Quiz Multi-Format Export Feature
**Context & Prompt for External Assistant (e.g., Claude/ChatGPT)**

You are deeply modifying a full-stack React + FastAPI application called **Swaya.me**, a platform for real-time polls, quizzes, and audience interaction. 
The user wants to add a **Multi-Format Export Feature** to download session responses in highly professional formats: **PDF, Word (.docx), PowerPoint (.pptx), and Excel (.xlsx)**.

The frontend is a React SPA (Vite, Ant Design, Redux) located in `/frontend/`.
The backend is a FastAPI application with SQLAlchemy (Async) located in `/backend/`.

## The Goal
In the host's Quiz History dashboard, they should be able to click an "Export Results" dropdown and download the quiz/poll results in any of the 4 formats. These files should be fully generated by the backend and streamed to the frontend.

## 2. Backend Implementation Breakdown

### A. Create `ExportServiceAsync`
Create a new file `backend/features/quiz/export_service_async.py`.
This service must contain a class `ExportServiceAsync` with the primary method:
`async def generate_export(self, db: AsyncSession, session_id: int, format_type: str) -> bytes | BytesIO:`

This function needs to:
1. Fetch the `QuizSession` by `session_id`, including its `Quiz`, `Questions`, `Participants`, and `Answers`.
2. Process the raw data into a structured format (e.g., lists of dictionaries).
3. Use a helper to generate a `BytesIO` buffer of the requested `format_type`.

**Format Logic Requirements:**
*   **Excel (`.xlsx`)**: Use `pandas` and `openpyxl`. Create a wide dataframe where rows are Participants and columns are Questions (with their answers).
*   **PDF (`.pdf`)**: Use `reportlab.platypus`. Create a title page, and for each question, draw a bar chart/pie chart using `matplotlib` (saved to temporary BytesIO) and embed it in the PDF document alongside absolute numbers.
*   **Word (`.docx`)**: Use `python-docx`. Add a Title. For each question, render the question text as a Heading, and embed the `matplotlib` chart image inline.
*   **PowerPoint (`.pptx`)**: Use `python-pptx`. Slide 1 should be a Title Slide. For every subsequent question, create a Blank layout slide, set the Slide Title to the Question Text, and embed the generated chart taking up the center of the slide.

## 3. Frontend Implementation Breakdown

### A. Update API Client
In `frontend/src/services/api.js`, locate `sessionAPI` and add:
```javascript

  api.get(`/quizzes/sessions/${sessionId}/export`, { 
    params: { format },
    responseType: 'blob' 
  }),
```

### B. Update `QuizHistory.jsx` UI
Modify `frontend/src/features/quiz/QuizHistory.jsx`.
1. Inside the `<SessionDetail>` component, locate the `<Card>` or the general layout where summary statistics are displayed.
2. Add a `Dropdown` button from `antd` labeled "Export Results".
3. The Dropdown menu items should be:
    * Export as Excel
    * Export as PDF
    * Export as Word
    * Export as PowerPoint
4. Write a handler `handleExport(format)`:
```javascript
const handleExport = async (format) => {
    try {
        const response = await sessionAPI.exportSessionResults(sessionId, format);
        // Create an object URL from the Blob response
        const url = window.URL.createObjectURL(new Blob([response.data]));
        const link = document.createElement('link'); // should be 'a' tag
        link.href = url;
        
        // Define extension based on format
        const extMap = { excel: 'xlsx', pdf: 'pdf', word: 'docx', ppt: 'pptx' };
        link.setAttribute('download', `session_results_${sessionId}.${extMap[format]}`);
        
        document.body.appendChild(link);
        link.click();
        link.parentNode.removeChild(link);
    } catch (err) {
        // Handle error (e.g. show Ant Design Message/Notification)
    }
}
```
